# Internationalisatie (i18n) — AI-vertaling met beheerinterface

Het inwonerportaal is meertalig. Vertalingen worden **automatisch gegenereerd** (vertaalmotor)
en kunnen door een **beheerder** worden bijgesteld via een beheerinterface. De inwoner kiest een
taal in het portaal; de keuze wordt als **standaardtaal per burger** server-side bewaard.

## Architectuur

```
Frontend (React + i18next)                Backend (Spring BFF)
  ├─ i18next-http-backend ──GET /api/i18n/{lng}──► I18nController ─► TranslationStore
  ├─ LanguageSwitcher ─────GET /api/i18n─────────►        (talenlijst)
  │                        PUT /api/voorkeuren/taal ─► VoorkeurenController ─► XworksClient (per burger)
  └─ BeheerI18n (?beheer=i18n) ─/api/beheer/i18n/**─► BeheerI18nController ─► TranslationStore ─► Translator
                                  (rol-gated: Beheerder)                         (MockTranslator → later Claude)
```

- **Bron = Nederlands** (`nl`), door ontwikkelaars beheerd in `i18n-seed/nl.json` (backend) en
  `src/locales/nl.json` (frontend, offline fallback). Sleutels zijn **plat/dotted** (`nav.home`).
- **Doeltalen** staan in de `TranslationStore` (BFF). Per sleutel: tekst + status
  `source` / `auto` / `reviewed` / `missing`.
- De frontend laadt bundels via **i18next-http-backend** uit `GET /api/i18n/{lng}`; de gebundelde
  JSON is alleen offline fallback.

## Vertaalmotor (Translator-poort)

- Nu: **`MockTranslator`** — markeert waarden als `"[<taal>] <bron>"` zodat de flow + beheer-UI
  werken en auto-regels zichtbaar "te reviewen" zijn.
- Later: **`ClaudeTranslator`** tegen de Anthropic Messages API (model `claude-haiku-4-5` /
  `claude-sonnet-4-6`). Implementeer `Translator`, maak hem `@Primary` (of zet de mock achter een
  profiel), en lees de API-key uit config. Zie de `claude-api`-skill voor de API-details.
- De motor vult **alleen ontbrekende** sleutels; door een beheerder gereviewde (`reviewed`)
  vertalingen worden **nooit overschreven**.

## Beheerinterface

Bereikbaar op **`/?beheer=i18n`** (alleen beheerders). Functionaliteit:
- Doeltaal kiezen; tabel met sleutel · NL-bron · vertaling (bewerkbaar) · status.
- **"Vertaal ontbrekende met AI"** — vult missende sleutels via de Translator (status `auto`).
- Handmatig bewerken + **Opslaan** → status `reviewed`.
- **Taal toevoegen / verwijderen** (ISO 639-1; `nl` is niet verwijderbaar).

### Toegang (rol-gate)
`Beheerder` bepaalt beheerderschap. **POC:** via header `X-Demo-Rol: beheerder` (de
frontend-beheer-calls sturen die mee); `inwonerportaal.beheer.demo-open=true` zet alles open.
**TODO (productie):** echte rol/claim uit Authentik/IdP (door de gatekeeper geïnjecteerd,
vertrouwd). Burgers zonder de rol krijgen **403**.

## Standaardtaal per burger

`GET/PUT /api/voorkeuren/taal` → `InwonerService` → `XworksClient.get/setVoorkeurstaal`. Dit is
**per-burger data** (reist mee met de BSN, server-side), in tegenstelling tot de vertalingen zelf
(applicatie-/beheerdata). De frontend past de opgeslagen voorkeur toe bij het laden; wisselen in de
switcher slaat de keuze op als standaard.

## Opslag

`FileTranslationStore` houdt alles in geheugen en persisteert naar
`${inwonerportaal.i18n.store-file:data/i18n-store.json}` (gitignored). Seedt bij een leeg bestand uit
`i18n-seed/`. **Later** vervangbaar door een database zonder de `TranslationStore`-poort te wijzigen.
Let op bij Docker: mount een volume als je edits over container-herbouw wilt behouden.

## Een taal toevoegen (handmatig, buiten de UI)

Drie plekken (de UI doet dit ook automatisch voor de store):
1. Backend store: via de beheer-UI of `POST /api/beheer/i18n/talen`.
2. Frontend label (optioneel, mooier in de switcher): `LABELS` in `components/LanguageSwitcher.tsx`.
3. Backend-validatie van de taalvoorkeur: `InwonerService.ONDERSTEUNDE_TALEN` (als de taal ook als
   *standaardtaal* instelbaar moet zijn).

## Verificatie (uitgevoerd)

Publieke bundels, talenlijst, rol-gate (burger → 403), taal toevoegen, AI-vullen (mock),
handmatige override → `reviewed`, idempotent her-vertalen (reviewed/auto blijven), en persistentie
naar schijf — alle live getest tegen het `stub`-profiel.
